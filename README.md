## Описание практических работ

### Практическая работа №3 — «10 релизов / SLA как контракт команды»

Файл: [`pr3_10_releases_sla.md`](./pr3_10_releases_sla.md)

- Определены ключевые SLA-метрики для интернет-магазина (web + mobile).
- Смоделировано 10 релизов с целевыми и фактическими SLA.
- Для каждого релиза описаны «чёрные лебеди» (непредвиденные события) и реакция команды.
- Проведён краткий анализ динамики выполнения SLA.

### Практическая работа №4 — «Передача между продуктом и техникой: релизы X и X+1»

Файл: [`pr4_handoff_sla.md`](./pr4_handoff_sla.md)

- Описаны две итерации — релиз X и релиз X+1.
- Построена схема взаимодействия ролей: Product Owner, Team Lead, QA, Support.
- Составлен документ «Передача требований и SLA».
- Описан инцидент с нарушением коммуникаций и принятые корректирующие меры.

### Практическая работа №5 — «Trade-off и метрики релиза»

Файл: [`pr5_tradeoff_metrics.md`](./pr5_tradeoff_metrics.md)

- Выполнен анализ задач релиза X с учётом ограничений по ресурсам.
- Построена матрица приоритетов (Value vs Effort, WSJF).
- Рассчитаны TCO, CPU, ROI, соотношение Run vs Grow.
- Добавлен «чёрный лебедь», выполнен пересчёт приоритетов и скорректирован план релизов.
